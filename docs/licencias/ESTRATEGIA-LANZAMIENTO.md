# Estrategia de lanzamiento — licencias (sin sobreingeniería)

**Estado:** plan operativo. **Activar** cuando el catálogo tenga masa crítica (varias obras publicables, no solo demos).  
**No** hardcodear descuentos en la calculadora web hasta entonces: se aplican **a mano** en presupuesto/email.

Canon de lista: `00-PRECIOS-REFERENCIA.md` · bolsillo: `TABLA-RAPIDA-PRECIOS.md`.

---

## 1. Cuándo se enciende

| Condición | Acción |
|-----------|--------|
| Catálogo “bueno” (varios temas reales, stems/preview OK) | Abrir ventana **founder / intro** |
| Primeros **5–10 clientes** de pago (no personal free) | Oferta documentada abajo |
| Tras 5–10 ventas o ~4 semanas de tracción | Cerrar founder → tarifa lista; revisar conversión |

Objetivo: **ventas reales + prueba social**, no regalar valor a agencias con budget.

---

## 2. Founder / intro (primeros 5–10 clientes)

**Nombre explícito en el correo/PDF:** *Founder price* o *Precio de lanzamiento* (no “descuento oculto”).

| Elemento | Qué ofrecer | Notas |
|----------|-------------|--------|
| **Descuento** | **−15 % a −25 %** sobre la tarifa de lista del uso | Ej. comercial 2y **169 €** → **~129–145 €** (elige un ancla y manténla en la ventana) |
| **Ancla ejemplo** | **129 €** comercial estándar 2 años (≈ −24 % s/ 169) | Alineado con “129 en vez de 149/169” de la crítica de mercado |
| **Micro** | Opcional: **69 €** en vez de 79 (−~13 %) o dejar 79 | Solo si el caso es sting/1 uso |
| **Stems** | **Gratis** en founder (valor lista +59 €) **o** crédito destacado | Una de las dos; no las dos a la vez salvo cliente muy estratégico |
| **Crédito** | `Music: [título] — Nimpo 3D Studio` en end cards / web / créditos | Pedido por escrito en la licencia |
| **Ads / exclusiva / buyout** | **No** bajar con founder salvo caso excepcional | El premium no se diluye en lanzamiento |

### Reglas

1. **Máximo 5–10** licencias founder (cuenta clientes de pago, no presupuestos enviados).  
2. Marca en el archivo interno: `FOUNDER #n · fecha · obra · precio cobrado`.  
3. Tras el cupo: **lista normal** (169 / 79 / 299…).  
4. NGO / amigo: no uses el cupo founder; usa la regla de descuento puntual de `00-PRECIOS`.

### Script de email (corto)

> Tarifa de lista comercial 2 años: **169 €**.  
> Como **founder price** de lanzamiento (cliente #X de 10): **129 €** + **stems incluidos** a cambio de crédito en la pieza y permiso a citar el uso como case study (sin secretos del proyecto).

---

## 3. Case studies (portfolio)

Cada venta founder (y las normales que autoricen) → **ficha breve** para web / redes / deck:

| Campo | Ejemplo |
|-------|---------|
| Título obra + cliente (o “productora indie”) | Despertar · corto festival |
| Uso | OST 1 tema, no exclusiva, 2 años |
| Resultado | Link o still (con permiso) |
| Quote del cliente | 1–2 frases (ver §4) |
| Qué se vendió | Comercial + stems / etc. |

**Permiso:** una línea en el mail de entrega: *“¿Podemos citar el proyecto (sin spoilers) en portfolio?”*  
Guardar: carpeta interna `case-studies/` o notas en el CRM/email; **no** publicar datos confidenciales.

**Cuándo en la web:** cuando haya 2–3 casos reales → sección o bloque en Sobre / ficha de obra (no inventar).

---

## 4. Testimonios y valoración de trabajos

**Pendiente de producto** (activar con catálogo + primeras ventas):

| Pieza | Qué | Notas |
|-------|-----|--------|
| **Testimonios** | Nombre + rol + 1–3 frases + obra | Solo con permiso escrito |
| **Valoración** | Estrellas o nota 1–5 + comentario corto | Evitar fake reviews; mejor 3 reales que 20 inventadas |
| **Dónde mostrar** | Ficha de obra, modal biblioteca, página Sobre, home | UI simple; no hace falta marketplace completo al inicio |
| **Cómo pedirlos** | Mail post-entrega + link o reply | Plantilla: *“¿Me dejas una frase de 2 líneas y una nota del 1 al 5?”* |
| **Credibilidad** | MIDI manual, stems, clearance directo | Repetir en copy junto al testimonio |

**No hacer aún:** sistema de reviews automáticas con login, gamificación, ni comprar reseñas.

**Sí hacer en el momento del lanzamiento founder:** checklist en cada cierre → pedir testimonio + permiso case study.

---

## 5. Checklist post-pago (founder o normal)

1. Cobro + PDF licencia.  
2. Entrega master (± stems).  
3. ¿Founder? → anotar cupo #n.  
4. Pedir: crédito + case study + **testimonio / valoración**.  
5. Si aceptan: guardar quote + nota en `case-studies` o doc interno.  
6. Cuando haya masa: publicar en web (bloque testimonios + 1–2 cases).

---

## 6. Qué no es esta estrategia

- No bajar la lista pública en código a 129 € “para siempre”.  
- No founder en exclusiva 3k / buyout 5.5k salvo decisión tuya caso a caso.  
- No sustituye A/B de precios tras 5–10 ventas a **tarifa lista**.
